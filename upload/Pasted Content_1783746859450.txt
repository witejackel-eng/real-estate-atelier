# 14. ANIMATION SYSTEM

Motion must feel premium, restrained, and reliable.

## Remove the blocking preloader

The current full-screen preloader should be removed unless it serves a real technical need.

Do not delay access to the website for a decorative logo animation.

If a transition is retained:

* It must be extremely short.
* It must never trap the page.
* It must not block navigation.
* It must not run on every route change.
* Content must remain available.

## Content visible by default

Never render essential page content permanently at `opacity: 0`.

Use progressive enhancement:

* Initial server-rendered content must be visible.
* Apply animation classes only after the client is ready.
* If JavaScript fails, the page remains fully usable.

## Motion language

Use motion for:

* Subtle heading entrances
* Image reveals
* Card image scale
* Underline transitions
* Mobile menu transitions
* Gallery transitions
* Filter drawer
* Accordion opening

Avoid:

* Animating every word individually throughout the site
* Repeated 40px upward movement on every block
* Excessive stagger delays
* Heavy parallax
* Scroll hijacking
* Animations that delay reading
* Motion that causes layout shift

## Reduced motion

Respect `prefers-reduced-motion`.

For reduced-motion users:

* Disable smooth scrolling
* Disable parallax
* Remove large translations
* Reduce or remove staggered motion
* Keep all content immediately visible

## Smooth scrolling

Only keep smooth scrolling if it does not:

* Break native keyboard navigation
* Interfere with anchor links
* Create scroll lag
* Degrade mobile performance
* Fight browser history restoration

Remove the smooth-scroll library if it provides no measurable value.
